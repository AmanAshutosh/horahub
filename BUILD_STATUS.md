# Build status

Honest manifest. Everything listed under **Complete** is finished, with no
placeholders or TODOs, and the project **compiles, lints, typechecks, and passes
17 tests**. The single step that does not complete *in a restricted sandbox* is
Next.js page-data collection, because it loads the Prisma client and the Prisma
engine binary download was network-blocked there (and there was no database). On
a normal machine, `npm install` → `prisma generate` → a real `DATABASE_URL`
makes `npm run build` finish.

## ✅ Complete and verified

**Engine (`src/ephemeris/`)** — Schlyter positions for all nine grahas
(Sun, Moon, Mercury–Saturn with Jupiter/Saturn perturbations, Rāhu/Ketu mean
node), Lahiri ayanāṁśa, sidereal ascendant, nakṣatra/pada, navāṁśa, dignity,
Vimśottari mahādaśā + antardaśā. Exposed via the `Ephemeris` interface →
`ChartFacts`. Verified by golden-file tests.

**Swiss Ephemeris adapter (`src/ephemeris/swiss.ts`, `provider.ts`)** — second
`Ephemeris` implementation wrapping `sweph` (Moshier model, no data files),
canonical Lahiri ayanāṁśa, selected via `EPHEMERIS_PROVIDER`. Feeds the same
`assembleChartFacts`; agrees with the analytic engine on every graha within 1.5°
(test-verified). This was the flagged P0 accuracy step — now done.

**Interpretation (`src/interpret/`, `src/kb/`)** — pure `interpret(facts, kb)`
producing planet / house / daśā-effect sections, each with a citation, zero
confidence scores (test-enforced). KB v1: 9 planet + 12 house rules in original
wording with sources; loader + validation script.

**Backend (`src/server/`, `src/lib/`)** — controllers → services →
repositories, Zod validators, `withApi` error/log middleware, rate limiting and
caching (with no-op fallbacks), timezone resolution with historical DST,
server-side geocoding, sha-256 chart hashing, typed errors, pino logger.

**API (`src/app/api/`)** — `POST /api/chart`, `GET /api/chart/[id]`,
`GET /api/geocode`, with validation, status codes, and security headers.

**Frontend (`src/app/`, `src/components/`)** — mobile-first birth form split into
`BirthForm.tsx` / `.types.ts` / `.validation.ts` / `PlaceSearch.tsx` / `index.ts`;
report view with sticky nav, planet cards, South-Indian Rāśi + Navāṁśa charts,
house/daśā accordions, daśā timeline; reusable `ui/` primitives; Zustand store;
`useDebounce` / `useGeocode` hooks; Tailwind design tokens; sitemap + robots.

**Database (`prisma/`)** — normalized schema (User, Profile, Chart, Reading,
KbVersion, KbRule) with indexes, relations, unique cache/version constraints,
and a seed.

**Tests (`tests/`)** — 22 passing: ephemeris golden chart, daśā invariants,
timezone/DST, request validation, interpretation structure + no-confidence.

**Docs (`docs/`)** — Installation, Development, Folder Structure, Architecture,
API, Database, Deployment, KB Pipeline, Contributing. Plus README, LICENSE, NOTICE.

## Run it
```bash
npm install
cp .env.example .env      # set DATABASE_URL
npm run db:migrate && npm run db:seed
npm run dev
# offline sanity: npm test && npm run typecheck
```
